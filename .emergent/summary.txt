<analysis>**original_problem_statement:**
The user wants to build TrustLens, an AI-powered relationship intelligence platform. The initial goal was to analyze behavioral changes and trust dynamics, providing clarity through pattern analysis without being accusatory. The project has evolved significantly into a structured behavioral intelligence tool with a sophisticated analysis pipeline.

**PRODUCT REQUIREMENTS:**
-   **Core Experience:** A user answers a series of questions, and the system generates a detailed analysis report.
-   **Branding & UI:** A cinematic, emotional, and analytical dark-themed UI. The landing page features a rotating hero image slideshow.
-   **Analysis Components:**
    -   **Suspicion Score:** A prominent, animated 0-100 score indicating pattern risk.
    -   **Dramatic Analysis Sequence:** An animated pre-reveal sequence to build anticipation.
    -   **Pattern Comparison Engine:** Compares user signals against a Relationship Case Database of ~300 synthetic cases, with demographic filtering capabilities.
    -   **AI Narrative Analysis:** The final report explanation is generated in real-time by an LLM (Claude Sonnet) based on the deterministic analysis results.
    -   **Perception Consistency Check:** Identifies and flags contradictory answers.
    -   **Relationship Timeline:** Tracks the user's Suspicion Score over multiple analyses.
-   **Features:**
    -   **Deep Analysis:** The full, multi-step investigation.
    -   **Relationship Pulse:** A quick, 3-5 question check-up.
    -   **Share Anonymized Report:** Generates a unique, private, shareable link to a simplified report.
    -   **PDF Export:** Allows users to download a full analysis report.
-   **Privacy:** The core flow is anonymous and session-based. No accounts are required.
-   **Architecture:** A hybrid model where core scoring and pattern matching are deterministic (backend logic), while the narrative and contextual follow-up questions are generated by an LLM.

**User's preferred language**: The user communicates in both English and French. The agent should default to English but be prepared to respond in French.
